Fractals ... again ...

L-Systems was presented in a previous segment - Fractal Trees. But Paul Bourke's site had so many systems, I couldn't resist myself from animating them. My first plan was to add them to the previous segment. The Problem was the rules for some of the systems would not run on the Turtle Graphics Engine implemented there.

So in this segment, we have an enhanced Turtle Graphics Engine and running it to revisit a lot of fractals explored in this site before. Plus a few brand new ones. 50+ fractals presented here. I will not go into what is L-System (previous segment has all that).

The enhanced Turtle Graphics Engine has the following features:

  • The Alphabet Set from the previous engine: F X + - [ ]
    • F: Draw a unit and move forward. The engine implemented earlier would draw a line. In this enhanced version, any shape can be passed into the engine to be drawn as a unit. Here almost all of the units are lines. The few which aren't will be highlighted in The Systems section.
    • X: Dummy
    • +: Rotate clockwise
    • -: Rotate anticlockwise
    • [: Save current state
    • ]: Restore previous state
  • New Alphabets added: f |
    • f: Move forward without drawing anything
    • |: Reverse direction
  • Provision to add Custom Alphabets and corresponding commands. Rules to render Sierpinski Triangle required a custom command.

The L-System is also enhanced to better rule the Turtle Graphics Engine. Along with the previous Rule, a chance Rule is also added where the replacement occurs only if the probability is satisfied (check out Random Tree).

    • X → F Y (75% chance to replace)

    • H is a custom command to reduce the length set in engine to 40%
    • ( is a custom command to store the current set length in the engine into the stack

  • Axiom: S F X
  • Rules:
    • X → f [ - - f ( H + + + f - - F X ) ] [ + + f ( H - - - f + + F X ) ]
    • S is a custom command to initiate a stack to store lengths in the engine
    • H is a custom command to reduce the length set in engine to L × sin(45°). Why? Using Law of sines to find the side of the next square.
