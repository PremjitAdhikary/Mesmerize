Binary Trees are the most basic and easiest of them all. A branching tree. Steps are as follows.

  • Draw a line
  • At the end, rotate to left and draw a shorter line and rotate to right and draw a shorter line. Play with the Angle for really weird ones.
  • Repeat previous step again and again for new lines. Play with the level for how long you want the tree to grow.

The Fractals demonstrated in previous examples are deterministic in nature. Which basically means that they will always produce the same tree. Or in other words, too precise to be natural. Random Tree is a non-deterministic (or stochastic in Mathematical terms) fractal.

Instead of just two branches, it can be anything from 2 to 5. The branches slider sets up a max though.

Next is that the length of each branch is a random factor of it's parent.

Aristid Lindenmayer developed a grammar-based system to model the growth patterns of plants. Coined as L-System, it has the following components:

  • Alphabet: The valid characters that can be included.
  • Axiom: A sentence made up with characters from the alphabet describing the initial state of the system.
  • Rules: These are applied to the axiom and then recursively applied to generate new sentences over and over.

A basic engine to render L-Systems.

Here we define the graphics engine for Alphabet 'F X + - [ ]'.

  • F: Draw a line and move forward
  • X: Dummy
  • +: Rotate anticlockwise
  • -: Rotate clockwise
  • [: Save current location
  • ]: Restore previous location

The Cantor Set from previous section can be written as a L-System as below.

Alphabet: A B
Axiom: A
Rules: (A → ABA) (B → BBB)

Here A translates to 'Draw a line forward' and B to 'Move forward without drawing'.

By Generations:

  • Generation 0: A
  • Generation 1: ABA
  • Generation 2: ABABBBABA
  • Generation 3: ABABBBABABBBBBBBBBABABBBABA

Pythagorean Tree is named so as each triple of touching squares encloses a right triangle.

  • Start with a square
  • On this 2 more squares are constructed. Each such that the 3 squares form a right triangle between them
  • Repeat for the 2 smaller squares

I was not sure if Phyllotaxis can be categorized as a fractal tree. It's more of a fractal flower. Meh...

The main idea is Phyllotactic spirals. Start from the center and keep rotating outwards. There are particular formulae for this to be rendered.

   ∅ = n × 137.5°
   r = c × √n

The above r and ∅ gives us the polar coordinates for the pattern to be rendered. Here as you increase petals, flower grows bigger and bigger.

While 137.5° is the correct angle, two other angles are also out there which gives out interesting patterns.
